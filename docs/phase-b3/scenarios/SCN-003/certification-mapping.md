## Parcours TEC.LOG

| Jalon | Condition |
|-------|-----------|
| Scénario validé | Score ≥ 60 + conformité + réappro avant GI |
| Quiz M1 | 60 % minimum (global M1) |
| Silver TEC.LOG | Quiz M1 + scénarios M1 requis |

## Preuves → certification

| Preuve session | Impact certification |
|----------------|-------------------|
| Déficit identifié + réappro | Démontre Prise de décision (Appliquer) |
| GI post-réappro uniquement | Séquence décisionnelle correcte |
| Score session ≥ 60 | Compte pour validation scénario M1 |
| Mode démo | Non officiel — pas d'impact certification |

## Compétences démontrées

| Action | Compétence |
|--------|------------|
| Détection déficit SO vs stock | Prise de décision |
| PO/GR corrective + putaway | Application procédure réappro |
| PICKING_M1 / GI post-stock | Exécution ordonnée |

## Écart double rôle (M1)

Mission Role (responsable d'opération) > Warehouse Role (opérateur entrepôt) — voir `01-dual-role-standard.md` § 6.

## Référence pilote

Aligné sur la logique de séquencement **SCN-005** ; SCN-003 transpose la discipline décisionnelle au cas rupture stock (moyen).
